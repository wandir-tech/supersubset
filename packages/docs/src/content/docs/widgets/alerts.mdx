# Alerts

The Alerts widget displays a list of operational alerts or notifications, color-coded by severity. Use it for monitoring dashboards, operations centers, or any scenario where users need to see a prioritized list of events.

<ExpandAll />

## Default Appearance

An alerts widget with default settings shows a list of alert cards with severity badges and timestamps.

## Configuration Options

### Title Field (`titleField`)

The field used for each alert's headline.

<ScreenshotComparison
  label="Title field binding"
/>

### Message Field (`messageField`)

The field used for each alert's description text.

<ScreenshotComparison
  label="Message field binding"
/>

### Severity Field (`severityField`)

The field that determines the color coding: `critical` (red), `warning` (amber), `info` (blue).

<ScreenshotComparison
  label="Severity color coding"
/>

### Timestamp Field (`timestampField`)

The datetime field shown as the alert's time.

<ScreenshotComparison
  label="Timestamp display"
/>

### Layout (`layout`)

Choose between `wrap` (cards flow horizontally and wrap) and `list` (vertical stack).

<ScreenshotComparison
  label="Layout: wrap vs list"
/>

### Max Items (`maxItems`)

Limit the number of alerts displayed. Useful for showing only the most recent or critical items.

<ScreenshotComparison
  label="Max items limit"
/>

### Show Timestamp (`showTimestamp`)

Toggle timestamp visibility.

<ScreenshotComparison
  label="Timestamp visibility"
/>

### Default Severity (`defaultSeverity`)

The severity used when the data doesn't include a severity field.

<ScreenshotComparison
  label="Default severity setting"
/>

### Empty State (`emptyState`)

What to show when there are no alerts: `placeholder` (friendly message) or `hidden` (widget disappears).

<ScreenshotComparison
  label="Empty state behavior"
/>
