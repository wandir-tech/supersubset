# Filter Scope

Filter scope determines which widgets are affected when a viewer changes a filter value. By default, filters are **global** — they affect all widgets. You can narrow the scope to target specific widgets or pages.

<ExpandAll />

## Scope Types

### Global Scope

The default. A global filter affects every widget on every page of the dashboard.

<ScreenshotComparison
  label="Global filter scope"
  designerSrc={filter_builder_designer}
  designerAlt="Global filter scope in designer mode"
/>

### Widget-Specific Scope

Limit a filter to affect only specific widgets. Useful when different sections of a dashboard use different datasets.

<FeatureScreenshot
  src={filter_bar_default_viewer}
  alt="Widget-specific scope"
  caption="Widget-specific scope"
/>

## Tips

- Start with **global scope** — it works for most dashboards
- Use **widget-specific scope** when you have widgets from different datasets on the same page
- Cross-filtering (click-based filtering) has its own scope rules — see [Cross-Filtering](/filters/cross-filtering/)
