# Cross-Filtering

Cross-filtering lets dashboard viewers click a chart element (a bar, pie slice, or data point) to filter all other widgets on the page. This creates an interactive exploration experience without explicit filter controls.

<ExpandAll />

## Default Appearance

When cross-filtering is configured, clicking a chart element highlights it and filters related widgets.

## Configuration Options

### Trigger Type (`trigger.type`)

The user action that activates the filter: `click` is the most common.

<ScreenshotComparison
  label="Trigger type configuration"
/>

### Source Widget (`trigger.sourceWidgetId`)

The widget whose elements can be clicked to trigger filtering.

<ScreenshotComparison
  label="Source widget selection"
/>

### Filter Field (`action.fieldRef`)

The field that gets filtered when the user clicks a chart element.

<ScreenshotComparison
  label="Filter field binding"
/>

## Tips

- Cross-filtering is most intuitive on [Bar Charts](/chart-types/bar-chart/) and [Pie Charts](/chart-types/pie-chart/)
- Users expect to click a bar/slice and see the rest of the dashboard update
- Combine with explicit [Select Filters](/filters/select-filter/) for a "Reset All" capability
- Keep the interaction intuitive — filter on the same field that labels the clicked element
