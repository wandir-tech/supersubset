# Select Filter

Select filters present a dropdown list of values that narrow dashboard data across all, or specific, widgets. Authors create them from **Dashboard Filters** in the designer header, then surface them through one or more Filter Bar widgets. Each bar can show all authored filters or only a selected subset.

## Configuration Options

### Field Reference (`fieldRef`)

The field the filter constrains when viewers apply a value. If the option source is field-backed, this same field can also provide the lookup values.

### Dataset Reference (`datasetRef`)

The dataset containing the field. This determines which data source the filter targets at runtime.

### Control Type (`type`)

Choose `Dropdown (single)` when viewers should pick one value at a time, or `Dropdown (multiple)` when the filter should emit an array of values.

### Option Source (`optionSource`)

Use `Authored static options` when the full choice list should travel with the dashboard schema. Field-backed option sources are reserved for hosts that resolve lookup values at runtime.

<FeatureScreenshot
  src={filter_builder_default_designer}

/>

### Operator (`operator`)

The comparison operator. Single-select filters commonly use `equals`; multi-select filters typically use `in`.

### Scope

Controls which widgets are affected by this filter. See [Filter Scope](/filters/filter-scope/) for details.

<FeatureScreenshot
  src={select_filter_category_viewer}
  alt="Placed filter bar widget with Category filter active"
  caption="Select filters affect only the widgets in their configured scope"
/>

### Filter Bar Placement

Select filters are authored once at the dashboard level. They only appear in viewer mode when a dashboard page includes at least one Filter Bar widget, and each Filter Bar can show every authored filter or a selected subset using the widget's `Shown Filters` setting.

## Tips

- Add one or more **Filter Bar** widgets wherever viewers should interact with filters
- Use the Filter Bar widget's **Shown Filters** setting to create all-filters and subset bars on the same page
- Use select filters for fields with **a limited set of values** (regions, categories, statuses)
- Use `Authored static options` when the choice list is known up front and should not depend on host lookups
- For date ranges, use a [Date Filter](/filters/date-filter/) instead
- Filters update all widgets in their scope in real time
